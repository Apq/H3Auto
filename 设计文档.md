# H3Auto 战场自动化 · 设计文档

> 本文档是 H3Auto 插件的设计与实现说明，以实际代码为准（`modules/*.inc.cpp`）。
> 设计方案的完整讨论见 `H3Note/H3Auto行动与目标策略实现方案.md`，本文档是其在代码仓库内的落地版。

---

## 1. 项目目标与定位

H3Auto 是一个英雄无敌3（SoD / HD Mod）战场自动化 DLL 插件。核心定位：

- **辅助人类玩家，减少重复性操作。** 在漫长战斗中代替玩家发出重复的游戏操作。
- **两种实现途径**：直接调用原版函数，或模拟玩家操作（点击 / 按键）。
- **只辅助人类玩家，不影响电脑阵营。** 回放、隐藏战斗、AI 回合一律不接管。

### 1.1 核心原则

1. **不接管全部决策。** 只在玩家预设了策略的部队上代为发出操作，其余保持手动。
2. **技能条件化。** 战争机器的可控性取决于英雄是否拥有对应技能（见 §2）。
3. **降级安全。** 配置的行动无法实现时，按玩家意愿降级为防御，或转回手动（见 §3.6）。
4. **只辅助人类。** 电脑阵营、回放、隐藏战斗一律不接管。
5. **配置仅驻留内存。** 5 套方案只存在进程内存，不写回 INI（INI 仅提供 UI 标签、热键等静态配置）。

### 1.2 不负责的范围

- 敌方部队 / 电脑阵营的策略。
- 战斗 AI 逻辑（由游戏原版处理）。
- 生物信息窗口扩展、战斗价值显示等非自动化功能。

## 2. 战争机器的条件化控制

战争机器平时由电脑自动操作（很笨）。仅在英雄**没有**对应技能时，才允许玩家通过插件做**有限设置**（仍不提供直接控制）：

| 战争机器 | 开放设置的条件 | 可设置的行动 |
| ---- | -------- | -------- |
| 弩车  | 英雄无**炮术**  | 远程攻击（目标选择器） |
| 箭塔  | 英雄无**炮术**  | 远程攻击（目标选择器） |
| 急救帐篷 | 英雄无**急救术** | 急救（目标选择器）   |
| 投石车 | 英雄**有弹道术** | 投石（攻城目标）    |

> 注意投石车方向相反：**有**弹道术时才开放设置（弹道术提升投石车效果，值得玩家干预）；其余三者是**无**对应技能时才开放（有技能时原版 AI 已够好）。

英雄技能查询：`HeroHasSkill`（弹道术 10 / 炮术 20 / 急救术 27）。

## 3. 需要提供的辅助能力

### 3.1 部队行动前英雄自动施法（快捷施法 / 循环施法）

- 每支部队行动前，英雄可先自动施放一个快捷法术。
- 由于每回合想施放的法术可能不同，支持**循环施法**：按回合轮换施放快捷施法的槽位。
- 循环施法只通过**快捷施法**功能实现（快捷施法本身最多 10 个槽位，快捷键 1-9/0）。循环列表的每个元素就是一个快捷键，最多排 10 个，按回合轮换。

### 3.2 自动防御

最简单的行动：本回合原地防御（`BA_DEFEND`）。

### 3.3 自动射击（远程攻击）

射手部队自动选择敌方目标射击。目标选择器支持：随机、远程速度优先、数量最多。

### 3.4 近身攻击

- 需要玩家设置两个位置：**站立格**（部队移动到哪）和**攻击格**（攻击哪个相邻敌人）。
- 两个位置都通过**点击进战场拾取**方式设置（见 §8.4）：点「站立格」按钮进入拾取点全战场任意格；点「攻击格」按钮进入拾取点站立格的相邻格。
- 攻击格只认站立格的相邻格；拾取时点非相邻格视为无效（忽略，不退出拾取）。

### 3.5 循环移动

- 玩家设置一个位置列表（最多 6 个点），部队按回合在这些点之间巡逻循环移动。
- 位置通过**点击＋追加**方式设置（见 §8.4）：点序列末尾「＋」进入战场拾取，左键逐个追加路径点，右键结束，满 6 个自动结束；点已选路径点可移除。

### 3.6 降级为自动防御

- 当设置的行动方式无法实现时（如近战攻击的目标不存在、移动路径被阻挡），若玩家勾选「允许降级」，则自动改为防御。
- 若玩家**未**勾选允许降级，则相当于该部队没有辅助（转为手动，玩家自行操作）。

## 4. 数据结构（源：`modules/ConfigLog.inc.cpp`）

### 4.1 枚举

```cpp
enum AutoActionKind {         // 行动类型
    AA_MANUAL = 0,            // 手动（不接管）
    AA_DEFEND = 1,            // 防御
    AA_WAIT = 2,              // 等待（保留占位，UI 已隐藏）
    AA_MOVE = 3,              // 循环移动
    AA_MELEE_ATTACK = 4,      // 近战攻击
    AA_RANGED_ATTACK = 5,     // 远程攻击
    AA_FIRST_AID = 6,         // 急救
    AA_CATAPULT = 7,          // 投石
};

enum AutoTargetSide {         // 目标阵营
    ATS_OWN = 0,              // 己方
    ATS_ENEMY = 1,            // 敌方
    ATS_EITHER = 2,           // 双方
};

enum AutoTargetSelector {     // 目标选择器（哪些项显示由 GetAllowedSelectors 控制）
    SEL_FIXED = 0,            // 固定部队
    SEL_RANDOM = 1,           // 随机
    SEL_SEQUENTIAL = 2,       // 顺序（按 army_slot_ix）
    SEL_NEAREST = 3,          // 最近
    SEL_FARTHEST = 4,         // 最远
    SEL_RANGED_SPEED = 5,     // 远程速度优先（现状：等价随机，待实现真实排序）
    SEL_COUNT_HIGH = 6,       // 数量最多
    SEL_COUNT_LOW = 7,        // 数量最少
    SEL_MOST_WOUNDED = 8,     // 伤势最重
    SEL_NONE = 9,             // 无（不额外指定）
};
```

> **枚举稳定性约定**：枚举值一经分配不再移动，新增项一律追加末尾。INI `[Actions]/[Sides]/[Selectors]` 段和默认标签数组严格按枚举下标排列；显示哪些项 / 顺序由 `GetAllowed*` 构建器控制。

### 4.2 规则结构

```cpp
struct AutoTargetRule {
    AutoTargetKind   kind;             // 目标类型（无 / 部队 / 位置）
    AutoTargetSide   side;             // 目标阵营
    AutoTargetSelector selector;       // 选择器
    int16_t          fixedSide;        // SEL_FIXED: 固定部队所属阵营
    int16_t          fixedSlot;        // SEL_FIXED: 固定部队槽位
    int16_t          fixedCreatureId;  // SEL_FIXED: 生物类型校验
    int16_t          position;         // AT_POSITION: 目标格
    int16_t          meleeStandHex;    // 近战/移动: 站立格 / 路径点1
    int16_t          meleeAttackHex;   // 近战: 攻击格
    int16_t          moveWaypoints[6]; // 循环移动: 路径点列表（最多 6）
    int8_t           moveWaypointCount;  // 有效路径点数
    int8_t           moveWaypointCursor; // 运行时巡逻游标
    bool             quickCastFirst;     // 行动前是否先快捷施法
    int8_t           spellSlots[10];     // 循环施法: 快捷键序列（1-9/0）
    int8_t           spellSlotCount;     // 有效槽位数
    int8_t           spellCursor;        // 运行时施法游标
};

struct AutoStackRule {
    AutoActionKind  action;        // 行动类型
    bool            allowDemote;   // 无法执行时是否降级为防御
    AutoTargetRule  target;        // 目标规则
};
```

### 4.3 方案存储

```cpp
AutoStackRule g_profiles[5][21];   // 5 套已确认方案 × 21 槽位，仅驻留内存
int           g_active_profile;    // 当前生效方案序号
AutoStackRule g_active_rules[21];  // 当前生效规则（执行端读取）
```

## 5. 行动可选集（源：`CellControl_GetAllowedActions`）

按部队类型返回允许的行动列表（决定行动下拉显示哪些项）：

- **普通近战部队**：手动、防御、循环移动、近战攻击。
- **射手部队**（`is_ranged`）：手动、防御、循环移动、近战攻击、远程攻击。
- **战争机器**：按 §2 的条件返回单一动作或手动（UI 以「启用/禁用」开关体现，见 §8.1）。

> `is_ranged` 由面板打开时按 `stack.info.shooter` 精确判定并存入 `CellData.is_ranged`；切换方案时复用，不用 `creature_type` 粗判（否则射手如大法师丢失远程选项）。

---

## 6. 目标可选集（源：`CellControl_GetAllowedSelectors`）

按行动类型返回允许的选择器（决定选择器下拉显示哪些项）：

- **远程攻击 / 近战攻击**：随机、远程速度优先、数量最多、无。
- **循环移动**：仅 `SEL_FIXED`（两点/多点式不需选择器菜单）。
- **急救**：己方伤员相关选择器。
- **投石**：攻城目标（无常规选择器）。

---

## 7. 规则规范化（源：`CellControl_NormalizeRule`）

行动变化或加载方案时调用，保证规则自洽：

1. 行动改变时，若当前选择器不在允许集，吸附到允许集首项。
2. 近战攻击：确保 `meleeStandHex/meleeAttackHex` 有默认值（站立格=当前位置，攻击格=相邻敌人）。
3. 循环移动：确保 `moveWaypoints` 已初始化，`kind=AT_POSITION`，`selector=SEL_FIXED`。
4. 切换到无目标的行动（防御 / 等待）时，清空目标相关字段。

## 8. 设置面板 UI（源：`modules/SettingsDlg.inc.cpp` + `CellControl.inc.cpp`）

### 8.0 模态方案：加固伪模态（不改真模态）

**背景**：曾考虑改为原版 `H3Dlg` 真模态窗口，但真模态靠嵌套阻塞消息循环独占输入，与战场拾取（拾取时需战斗界面响应点击）机制冲突——真模态期间战场被冻结，点不动；要拾取只能关窗重开，体验更差。

**结论**：保留当前**伪模态**方案（自绘到 `screenPcx16` + 自建 hook `Hook_BattleMsgProc`），重点加固：

- 拾取期间确保背景战斗界面不响应误点（屏蔽 hover / 点击穿透）。
- 模态遮挡干净（`ForcePanelModalDepth_` 顶模态计数器）。
- 重绘时机正确（拾取进入 / 退出、格子展开 / 收起边界）。

### 8.1 布局：一行一格宽卡片（568×72）

单列纵向排列，每支部队一张宽卡片。可见约 4~5 行，超出用滚动条。

**普通部队卡：**

```text
[图标]  快捷施法 ☑ [1▾][2▾]…＋   行动 [远程攻击▾]      选择器 [数量最多▾]
 58×64  数量 12                    允许降级 ☑            （近战: 站立格/攻击格）
        位置 A07                                        （移动: 路径点序列＋）
```

**战争机器卡**（弩车 / 箭塔 / 帐篷 / 投石，仅技能条件满足时可设）：

```text
[图标]  数量 1        启用 ☑          选择器 [数量最多▾]
 58×64  位置 K08     （固定动作说明）  （投石: 攻城目标）
```

战争机器卡：隐藏施法两行、隐藏降级复选框；用「启用/禁用」开关代替行动下拉（启用=按选择器接管它唯一能做的事，禁用=交回原版 AI）；位置虽固定但保留显示供参考。

### 8.2 普通部队卡小列布局

- **图标小列**（x=4，58×64）：部队 DEF 图标 + 金框。
- **位置/数量小列**（x=68，宽 74）：自上而下——快捷施法复选框、选择魔法（槽位序列＋）、数量（右对齐）、位置（贴格子下边缘左对齐）。
- **行动小列**（x=148，宽 202）：行动下拉（上）+ 允许降级复选框（下）。
- **选择器小列**（x=358，宽 202）：选择器下拉（上）+ 近战站立/攻击格 或 移动路径序列（下）。

### 8.3 下拉展开几何

- 各下拉从其所在小列的行下边缘展开，宽度取所在小列宽（魔法槽位下拉用窄列宽）。
- `GetExpandListLeftX / GetExpandListTopY / GetExpandListWidth` 统一计算基准，绘制（`DrawExpandedTo`）、命中（`HitExpandVisibleIndex`）共用。

### 8.4 战场拾取（近战格 / 移动路径）

统一为「点击进战场拾取」交互，复用原版 `0x464380` 屏幕坐标→hex 转换：

- **近战站立/攻击格**：单发拾取——点按钮进拾取，点一个格回填一个。攻击格只认站立格相邻格，点非相邻格无效（忽略不退出）。**（待改：当前主用坐标下拉，需切换到战场拾取并接线激活）**
- **循环移动路径**：连续追加——点「＋」进拾取，左键逐个追加（最多 6），右键结束，满 6 自动结束；点已选点移除。
- 拾取期间面板挂起（`s_move_pick_cell` 等状态），hook 捕获战场点击转 hex 回填，实时重画。

### 8.5 标签来源（`LoadLabels_`）

INI `[Actions]/[Sides]/[Selectors]` 段提供 UI 标签，严格按枚举下标排列。缺失时用内置默认标签数组兜底。

## 9. 方案保存流程

目标行为：打开界面自动加载上次设置；调整过程中不影响已保存设置；仅点「确定（勾✓）」才真正写入。

- 打开面板：`g_profiles → s_p.draft_rules`，进入草稿副本编辑，原方案不动。
- 编辑格子：改动只写 `s_p.cells[].data.rule`（内存草稿）。
- 切换方案（`SelectProfile_`）：先 `SaveCurrentCellsToDraft_` 存回草稿，再 `LoadSelectedProfileIntoCells_` 载入目标方案。
- 点确定（`CommitAndCloseSettingsPanel_`）：`SaveCurrentCellsToDraft_` → `CommitProfiles` 一次性提交全部 5 套到 `g_profiles`。
- 点取消 / ESC：直接关闭，草稿丢弃。

### 9.1 设置生命周期（锚定「一场战斗」）

设置只对**当前这场战斗**有效，但生命周期以「玩家接受战斗结果」为终点，而非战斗动画打完：

1. 设置在战斗开始时初始化，整场战斗期间常驻内存。
2. **重打不算新战斗**：战斗结束弹结果窗，玩家点「取消/重打」→ 重打同一场，5 套方案原样保留，绝不能丢。
3. 反复重打（自动战→取消→手动战→取消…）仍是同一场，设置一路保留。
4. **只有玩家点「接受/确定」战斗结果**，这场战斗才真正结束，此时才清除内存中的设置。
5. 底层理由：不接受结果就能一直重打（除非读档，而读档是另一次战斗上下文），所以设置必须留着；一旦接受就再也回不到这场战斗，设置才可以安全清除。

## 10. 执行分发（源：`modules/AutoExecute.inc.cpp`）

### 10.1 接管判定（`DecideTakeover`）

回放/隐藏战斗（`IsHiddenBattle`）一律 `CD_KEEP_ORIGINAL`。否则：

| 单位    | 判定                        |
| ----- | ------------------------- |
| 弹药车   | 保持原版                      |
| 急救帐篷  | 配了急救→执行；否则有急救术→原版，无→交 AI  |
| 投石车   | 配了投石→执行；否则有弹道学→原版，无→交 AI  |
| 弩车/箭塔 | 配了远程→执行；否则有炮术→原版，无→交 AI   |
| 普通部队  | 手动→保持原版；否则按配置执行（绝不交回 AI） |

### 10.2 动作提交（`TrySubmitConfiguredAction_`）

按 `rule.action` 分发到 `Submit*`；普通部队失败且允许时降级 `SubmitDefend_`：

- **防御** `SubmitDefend_`：`action=BA_DEFEND`。
- **等待** `SubmitWait_`：`action=BA_WAIT`。
- **循环移动** `SubmitMove_`：**现状**仍是旧的单目标移动（`ResolvePositionTarget_` → `BA_WALK`），**待改**为按 `moveWaypoints` + `moveWaypointCursor` 逐点巡逻循环。
- **近战** `SubmitMelee_`：攻击格上有敌人才出手；`move_type=7`，`action=BA_WALK_ATTACK`，param=站立格，target=攻击格。
- **远程** `SubmitRanged_`：`SelectStackTarget_` 选敌方目标，`action=BA_SHOOT`。
- **急救** `SubmitFirstAid_`：选己方伤员，`action=BA_FIRST_AID`。
- **投石** `SubmitCatapult_`：城墙位置，`action=BA_CATAPULT`。

写入后原版主循环（`FUN_004786b0` 等）接管动画 / 伤害 / 回合推进。

### 10.3 目标选择（`SelectStackTarget_` / `PickBySelector_`）

先按 `side` 过滤双方 `stack[2][21]` 得候选，再按选择器挑一个：

- `SEL_NEAREST/FARTHEST`：按 `hex_ix` 距离。
- `SEL_COUNT_HIGH/LOW`：按存活数量。
- `SEL_MOST_WOUNDED`：按损失生命 `(count_at_start-count_current)*1000 + lost_hp`。
- `SEL_SEQUENTIAL`：按 `army_slot_ix`。
- `SEL_RANGED_SPEED / SEL_RANDOM / 其它`：**现状**取随机候选。
- `SEL_FIXED`：固定部队（`fixedSide/fixedSlot/fixedCreatureId`）。

### 10.4 快捷施法执行

**现状**：`quickCastFirst / spellSlots` 尚未接执行端。目标：部队行动前，若勾选则按 `spellSlots` + `spellCursor` 用英雄快捷施法槽位施放法术（每回合轮换），再执行本部队的主动行动。

5. **`SEL_RANGED_SPEED` 真实排序**：远程优先 → 速度降序。
6. **移动寻路可达性校验**：当前固定位置优先，缺完整原版可达性校验。
7. **降级语义边界**：确认「未允许降级」时是否完全交回原版手动（而非静默跳过）。
8. **设置清除时机**：把内存方案的清除挂在原版「战斗结果窗口」接受/重打的分支 hook 点上（真正实现时再逆向定位；重打不清、接受才清）。

## 14. 构建与部署

- **统一入口**：`build_and_deploy.bat`（编译 + 部署一步完成，不要单独跑 `build.bat` / `deploy.bat`）。
- 源文件 UTF-8，中文游戏字体走 GBK；换行 CRLF。
- 资源尺寸严格校验：卡片 / 金框尺寸改动后必须用 `tools/gen_*.js` 重生对应 PCX，否则背景加载失败。
- DLL 在游戏进程启动时加载，改动后必须**完全重启游戏**才生效。
