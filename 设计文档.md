# H3Auto 设计文档

## 目标

在手动战斗中，在轮到人类玩家行动时，按玩家预设的自动化策略自动执行己方操作，减少重复操作。

## 功能范围

1. **设置界面**：右键点击战场"自动战斗"按钮，弹出模态窗口，配置己方每支部队的自动化策略。
2. **自动化执行**：轮到人类玩家行动时，按设置自动执行（默认全部手动）。
3. **快捷键**：启动/停止自动化。

H3Auto 不负责：
- 敌方部队的策略设置
- 战斗 AI 逻辑（由游戏原版处理）
- 生物信息窗口扩展（MegaDesc）
- 战斗价值显示（H3BattleValueInfo）

## 设置界面

### 布局说明

- **7行 × 3列 = 21格**，每格一支己方部队
- 每格**上下两部分**：
  - **上部**：部队图标（DEF 静态，36×36），图标下方左侧战区坐标（如 A01），右侧数量（×N）
  - **下部**：下拉框，选择行动策略
- 每行3格从左到右排列，满3个换下一行
- 不足7支时后面的行留空

### 控件结构

| 区域 | 控件类型 | 说明 |

| 背景 | PCX16 | 美化背景图 |

| 每格-图标 | DEF 静态 | 显示生物图标 |
| 每格-位置文本 | 文本 | 图标下方左侧，战区坐标（如 A01~H15） |
| 每格-数量文本 | 文本 | 图标下方右侧，×N |

| 取消按钮 | 按钮 | 关闭，不保存 |
| 确定按钮 | 按钮 | 保存，关闭 |

**窗口尺寸估算**：
- 每格宽 ≈ 160px，3列 × 160 = 480px，加上内边距约 520px
- 每格高 ≈ 54px（上36图标+下18文字+下拉框），7行 ≈ 378px，加上标题和按钮约 460px
- 总计约 580×480px

### 行动策略选项

| 值 | 文本 | 说明 |
|---|---|---|
| 0 | 手动 | 默认，不干预，用户自行操作 |
| 1 | 防御 | DEFEND，不要主动攻击 |
| 2 | 随机射击 | 随机选择目标生物攻击 |
| 3 | 智能射击 | 优先敌方远程 > 速度快 > 生物等级高 |
| 4 | 平衡射击 | 打击使敌方剩余数量趋于平衡（打数量最多的那队） |

> **下拉框实现**：H3API 无内置下拉框控件。用 `_DlgCustomButton_` 按钮 + 弹出小列表框模拟，点击按钮时展开选项列表，选中后收起并更新按钮文本。

### 按钮事件

- **取消**：关闭窗口，不保存。
- **确定**：保存设置到内存，关闭窗口。

## 自动化执行

### 触发时机

在 `Hook_CycleCombatScreen @ 0x495C50`（战斗动画循环）每帧检测。

### 判断条件

```cpp
bool IsHumanPlayerTurn() {
    if (!o_BattleMgr || o_BattleMgr->auto_combat) return false;
    _BattleStack_* s = o_BattleMgr->active_stack;
    if (!s) return false;
    int side = o_BattleMgr->current_active_side;
    _Hero_* h = o_BattleMgr->hero[side];
    return h != nullptr;  // 有英雄 = 人类控制
}
```

### 执行策略

```cpp
void ExecuteStrategy(_BattleStack_* stack) {
    int strategy = GetStrategy(stack);
    switch (strategy) {
        case 0: return;  // 手动，不干预
        case 1: SetAction(stack, DEFEND);  break;         // 防御
        case 2: RandomShot(stack);       break;         // 随机射击
        case 3: SmartShot(stack);   break;         // 智能射击
        case 4: BalancedShot(stack); break;         // 平衡射击
    }
}

void RandomShot(_BattleStack_* stack) {
    // 从敌方存活部队中随机选一个目标
    auto targets = GetEnemyStacks(stack->side);
    if (targets.empty()) return;
    int idx = rand() % targets.size();
    SetTarget(stack, targets[idx]);
}

void SmartShot(_BattleStack_* stack) {
    // 优先敌方远程 > 速度快 > 生物等级高
    auto targets = GetEnemyStacks(stack->side);
    if (targets.empty()) return;
    // 第一排序：是否远程（远程优先）
    // 第二排序：速度（高优先）
    // 第三排序：生物等级（高优先）
    std::sort(targets.begin(), targets.end(), [](auto& a, auto& b) {
        if (a.is_ranged != b.is_ranged) return a.is_ranged > b.is_ranged;
        if (a.speed != b.speed)        return a.speed > b.speed;
        return a.level > b.level;
    });
    SetTarget(stack, targets[0]);
}

void BalancedShot(_BattleStack_* stack) {
    // 打击使敌方剩余数量趋于平衡——打数量最多的那队
    auto targets = GetEnemyStacks(stack->side);
    if (targets.empty()) return;
    // 按剩余数量降序排列，选最多的那队
    std::sort(targets.begin(), targets.end(), [](auto& a, auto& b) {
        return a.count > b.count;
    });
    SetTarget(stack, targets[0]);
}
```

> **关键**：如何"自动执行动作"——目前未确认游戏内部是否有机制让插件向战斗系统提交动作。可能的路径：直接修改 `active_stack` 的状态字段、或 Hook 战斗行动提交函数注入动作。需要进一步逆向确认。

## 数据结构

### 策略配置

```cpp
// 每支部队的策略（per-battle）
struct StackStrategy {
    int stack_index;   // 在 stack[] 数组中的索引
    int strategy;      // 0=手动, 1=防御, 2=随机射击, 3=智能射击, 4=平衡射击
};

// 全局状态
struct H3AutoState {
    bool enabled;                       // 自动化是否启用（DisableOnStart=0 时默认 true）
    StackStrategy strategies[21];      // 己方最多21个槽（10主力+增援等）
};
```

### INI 配置

```ini
[General]
DisableOnStart=0      ; 战斗开始时是否禁用自动化：0=不禁用（默认启用），1=禁用

[Hotkey]
StartAutomation=0x78   ; F7
StopAutomation=0x79     ; F8
```

## Hook 列表

| 地址 | 类型 | 用途 |
|---|---|---|
| `0x495C50` | HiHook | 战斗动画循环，检测行动时机并执行策略 |
| `0x41B120` | LoHook | 对话框 DefProc，拦截右键点击弹出设置窗口 |
| `0x464F10` | HiHook | 施法后标记策略状态变更 |
| `0x4781C0` | HiHook | 战斗开始，清空策略状态 |

## 实现顺序

3. **设置窗口**：右键拦截 -> 弹出窗口 -> 下拉框实现 -> 控件交互
4. **自动化执行**：行动时机检测 -> 策略执行（待逆向确认）
5. **INI 配置**：读写热键和选项
