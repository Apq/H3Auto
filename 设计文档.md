# 战场自动化 设计文档

## 目标

在手动战斗中，在轮到人类玩家行动时，按玩家预设的自动化策略自动执行己方操作，减少重复操作。

## 功能范围

1. **设置界面**：右键点击战场"自动战斗"按钮，弹出模态窗口，配置己方每支部队的自动化策略。
2. **自动化执行**：轮到人类玩家行动时，按设置自动执行（默认全部手动）。
3. **快捷键**：启动/停止自动化。

战场自动化 不负责：
- 敌方部队的策略设置
- 战斗 AI 逻辑（由游戏原版处理）
- 生物信息窗口扩展（MegaDesc）
- 战斗价值显示（H3BattleValueInfo）

### 布局说明

- **4列 × 3行 = 12格**，每格一支己方部队
- 每格**上下三部分**：
  - **最上**：部队图标（DEF 静态，36×36），图标下方左侧战区坐标（如 A01），右侧数量（×N）
  - **中部**：行动策略下拉框
  - **下部**：目标策略下拉框
- 每行4格从左到右排列，满4个换下一行
- 不足12支时后面的格留空
- 超过12支时，第4行之后需要滚动条（或分页，每页最多3行）

**窗口尺寸估算**：
- 每格宽 ≈ 150px，4列 × 150 = 600px，加上内边距约 660px
- 每格高 ≈ 70px（上36图标+下34文字+2个下拉框），3行 ≈ 210px，加上标题和按钮约 280px
- 总计约 700×320px，加上内边距约 700×440px

### 行动策略选项

| 值 | 文本 | 说明 |
|---|---|---|
| 0 | 手动 | 默认，不干预，用户自行操作 |
| 1 | 防御 | DEFEND，不要主动攻击 |
| 2 | 近战攻击 | 主动向敌方移动并攻击 |
| 3 | 随机射击 | 随机选择目标生物攻击 |
| 4 | 顺序射击 | 按敌方站位顺序依次射击（替代原"智能射击"） |
| 5 | 循环移动 | 循环移动到不同位置（绕后、包抄等） |

### 目标策略选项

| 值 | 文本 | 说明 |
|---|---|---|
| 0 | 无 | 默认，不指定特殊目标优先级 |
| 1 | 指定位置 | 指定攻击特定位置的目标 |
| 2 | 远程和高速优先 | 优先攻击敌方远程部队，速度高的优先 |
| 3 | 数量优先 | 优先攻击数量最多的敌方部队 |

> **下拉框实现**：H3API 无内置下拉框控件。用 `_DlgCustomButton_` 按钮 + 弹出小列表框模拟，点击按钮时展开选项列表，选中后收起并更新按钮文本。

> **战区坐标**：用 A01~H15 格式表示，A=左列，H=右列，01=上行，15=下行。中线为 D/E 列分界。

### 按钮事件

- **取消**：关闭窗口，不保存。
- **确定**：保存设置到内存，关闭窗口。

## 自动化执行

### 触发时机

在 `Hook_CycleCombatScreen @ 0x495C50`（战斗动画循环）每帧检测。

### 判断条件

```cpp
bool IsHumanPlayerTurn() {
    if (!o_BattleMgr || o_BattleMgr->auto_combat) return false;
    _BattleStack_* s = o_BattleMgr->active_stack;
    if (!s) return false;
    int side = o_BattleMgr->current_active_side;
    _Hero_* h = o_BattleMgr->hero[side];
    return h != nullptr;  // 有英雄 = 人类控制
}
```

### 执行策略

```cpp
void ExecuteStrategy(_BattleStack_* stack) {
    int action = GetActionStrategy(stack);
    int target = GetTargetStrategy(stack);

    // 目标选择（受目标策略影响）
    auto targets = GetEnemyStacks(stack->side);
    if (targets.empty()) return;

    switch (target) {
        case 0: break;  // 无特殊目标，按默认或行动策略选
        case 1: targets = FilterByPosition(targets, specified_pos); break;
        case 2: SortByRangedAndSpeed(targets); break;  // 远程和高速优先
        case 3: SortByCount(targets); break;             // 数量优先
    }

    switch (action) {
        case 0: return;                    // 手动，不干预
        case 1: SetAction(stack, DEFEND);  break;         // 防御
        case 2: SetAction(stack, ATTACK, targets[0]);     // 近战攻击
        case 3: SetTarget(stack, RandomPick(targets));     // 随机射击
        case 4: SetTarget(stack, SequentialPick(targets)); // 顺序射击
        case 5: SetAction(stack, MOVE_CYCLE);             // 循环移动
    }
}
```

> **关键**：如何"自动执行动作"——目前未确认游戏内部是否有机制让插件向战斗系统提交动作。可能的路径：直接修改 `active_stack` 的状态字段、或 Hook 战斗行动提交函数注入动作。需要进一步逆向确认。

## 数据结构

### 策略配置

```cpp
// 每支部队的策略（per-battle）
struct StackStrategy {
    int stack_index;       // 在 stack[] 数组中的索引
    int action_strategy;   // 行动策略：0=手动, 1=防御, 2=近战攻击, 3=随机射击, 4=顺序射击, 5=循环移动
    int target_strategy;   // 目标策略：0=无, 1=指定位置, 2=远程和高速优先, 3=数量优先
    int target_position;   // 当 target_strategy=1 时的指定位置（战区坐标索引 0~39）
};

// 全局状态
struct H3AutoState {
    bool enabled;                       // 自动化是否启用（DisableOnStart=0 时默认 true）
    StackStrategy strategies[21];       // 己方最多21个槽（10主力+增援等）
};
```

### INI 配置

```ini
[General]
DisableOnStart=0      ; 战斗开始时是否禁用自动化：0=不禁用（默认启用），1=禁用

[Hotkey]
StartAutomation=0x78   ; F7
StopAutomation=0x79     ; F8
```

## Hook 列表

| 地址 | 类型 | 用途 |
|---|---|---|
| `0x495C50` | HiHook | 战斗动画循环，检测行动时机并执行策略 |
| `0x41B120` | LoHook | 对话框 DefProc，拦截右键点击弹出设置窗口 |
| `0x464F10` | HiHook | 施法后标记策略状态变更 |
| `0x4781C0` | HiHook | 战斗开始，清空策略状态 |

## 实现顺序

1. **框架搭建**：项目结构、ConfigLog、Entry
2. **快捷键**：注册 F7/F8
3. **设置窗口**：右键拦截 -> 弹出窗口 -> 下拉框实现 -> 控件交互
4. **自动化执行**：行动时机检测 -> 策略执行（待逆向确认）
5. **INI 配置**：读写热键和选项
