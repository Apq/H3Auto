# 打铁助手（H3Auto）· 设计文档

> 本文档是「打铁助手」插件的设计与实现说明，以实际代码为准（`modules/*.inc.cpp`）。
> 设计方案的完整讨论见 `H3Note/H3Auto行动与目标策略实现方案.md`，本文档是其在代码仓库内的落地版。

---

## 1. 项目目标与定位

打铁助手（工程名 H3Auto）是一个英雄无敌3（SoD / HD Mod）战场自动化 DLL 插件。核心定位：

- **辅助人类玩家，减少重复性操作。** 在漫长战斗中代替玩家发出重复的游戏操作。
- **两种实现途径**：直接调用原版函数，或模拟玩家操作（点击 / 按键）。
- **只辅助人类玩家，不影响电脑阵营。** 回放、隐藏战斗、AI 回合一律不接管。

### 1.1 核心原则

1. **不接管全部决策。** 只在玩家预设了策略的部队上代为发出操作，其余保持手动。
2. **技能条件化。** 战争机器的可控性取决于英雄是否拥有对应技能（见 §2）。
3. **降级安全。** 配置的行动无法实现时，按玩家意愿降级为防御，或转回手动（见 §3.6）。
4. **只辅助人类。** 电脑阵营、回放、隐藏战斗一律不接管。
5. **配置仅驻留内存。** 5 套方案只存在进程内存，不写回 INI（INI 仅提供 UI 标签、热键等静态配置）。

### 1.2 不负责的范围

- 敌方部队 / 电脑阵营的策略。
- 战斗 AI 逻辑（由游戏原版处理）。
- 生物信息窗口扩展、战斗价值显示等非自动化功能。

## 2. 战争机器的条件化控制

战争机器平时由电脑自动操作（很笨）。插件按机器类型决定是否进面板，以及「手动」是否可选（仍不提供原版之外的直接操控）：

| 战争机器 | 开放设置的条件 | 可设置的行动 |
| ---- | -------- | -------- |
| 弩车  | 始终可进面板 | **有炮术**：手动（默认）/ 远程攻击；**无炮术**：仅远程攻击 |
| 箭塔  | 始终可进面板 | 同弩车 |
| 急救帐篷 | 始终可进面板 | **有急救术**：手动（默认）/ 急救治疗；**无急救术**：仅急救治疗 |
| 投石车 | 英雄有**弹道术** | **手动 / 防御**（不做攻城目标自动化） |
| 弹药车 | **永不进面板** | 不接管（始终原版） |

> 弩车/箭塔：炮术决定是否允许「手动」；无炮术时强制自动远程。
> 急救帐篷：急救术决定是否允许「手动」；无急救术时强制自动急救。
> 投石车：无弹道术则不出现在面板。

英雄技能查询：`HeroHasSkill`（弹道术 10 / 炮术 20 / 急救术 27）。

## 3. 需要提供的辅助能力

### 3.1 部队行动前英雄自动施法（快捷施法 / 循环施法）

- 每支部队行动前，英雄可先自动施放一个快捷法术。
- 由于每回合想施放的法术可能不同，支持**循环施法**：按回合轮换施放快捷施法的槽位。
- 循环施法只通过**快捷施法**功能实现（快捷键 1-9/0）。UI 单行最多 **10 个**快捷键槽位（`spellSlots[10]`），运行游标保存在本场部队跟踪表；原版快捷施法仍有 10 个键可选。

### 3.2 自动防御

最简单的行动：本回合原地防御（`BA_DEFEND`）。

### 3.3 自动射击（远程攻击）

射手部队自动选择敌方目标射击。目标选择器支持：随机、远程速度优先、数量最多。

### 3.4 循环近战

- `AA_MELEE_ATTACK` 的枚举值固定为 `4`，显示名改为「循环近战」。
- 每支部队最多设置 10 组 **站立位＋攻击位**；执行时按游标逐组循环，只有动作成功提交后才推进游标。
- 「＋」用于追加组合；已有组合显示为 `A01→A02`，点击任意组合可连续重选并覆盖该记录。
- 每组采用一次连续战场拾取：先点**站立格**，插件立即用原版 `CCellShd` 蓝色格子在战场临时标示；再点与其不同且相邻的**攻击目标格**。第二格相同或不相邻时忽略并继续拾取。
- 第二格成功、右键取消或面板关闭时，临时标示随战场重绘撤销；整个拾取过程点击均由 BattleUI 输入屏障吞掉，不会触发真实部队行动。

### 3.5 循环移动

- 玩家设置一个位置列表（最多 16 个点），部队按回合在这些点之间巡逻循环移动。
- 位置通过**点击＋追加**方式设置（见 §8.4）：点序列末尾「＋」或已有点进入战场拾取 1 格，写回对应槽（最多 16）；右键取消不改原记录。

### 3.6 降级为自动防御

- 当设置的行动方式无法实现时（如循环近战当前组合的目标不存在、移动路径被阻挡），若玩家勾选「允许降级」，则自动改为防御。
- 若玩家**未**勾选允许降级，则相当于该部队没有辅助（转为手动，玩家自行操作）。

## 4. 数据结构（源：`modules/ConfigLog.inc.cpp`）

### 4.1 枚举

```cpp
enum AutoActionKind {         // 行动类型
    AA_MANUAL = 0,            // 手动（不接管）
    AA_DEFEND = 1,            // 防御
    AA_WAIT = 2,              // 等待（保留占位，UI 已隐藏）
    AA_MOVE = 3,              // 循环移动
    AA_MELEE_ATTACK = 4,      // 循环近战（枚举值保持不变）
    AA_RANGED_ATTACK = 5,     // 远程攻击
    AA_FIRST_AID = 6,         // 急救
};

enum AutoTargetSide {         // 目标阵营
    ATS_OWN = 0,              // 己方
    ATS_ENEMY = 1,            // 敌方
    ATS_EITHER = 2,           // 双方
};

enum AutoTargetSelector {     // 按行动分流
    SEL_RANDOM = 0,           // 随机（远程/急救）
    SEL_RANGED_SPEED = 1,     // 远程：远程速度优先
    SEL_COUNT_HIGH = 2,       // 远程：数量最多
    SEL_WOUND_RATIO = 3,      // 急救：失血比例最高
    SEL_WOUND_VALUE = 4,      // 急救：失血数值最大
};
```

> INI `[Actions]/[Selectors]` 段和默认标签数组按枚举下标排列。目标阵营下拉已停用，阵营由各行动固定。

### 4.2 规则结构

```cpp
struct AutoTargetRule {
    AutoTargetKind   kind;             // 目标类型（无 / 部队 / 位置）
    AutoTargetSide   side;             // 目标阵营
    AutoTargetSelector selector;       // 选择器
    int16_t          meleeStandHex;    // 旧版单组近战兼容镜像
    int16_t          meleeAttackHex;   // 旧版单组近战兼容镜像
    int16_t          moveWaypoints[16]; // 循环移动: 路径点列表（最多 16）
    int8_t           moveWaypointCount;  // 有效路径点数
    int16_t          meleeStandHexes[10]; // 循环近战: 各组站立位
    int16_t          meleeAttackHexes[10];// 循环近战: 各组攻击位
    int8_t           meleePairCount;     // 有效组合数
};

struct AutoStackRule {
    AutoActionKind  action;              // 行动类型
    AutoTargetRule  target;              // 目标规则
    bool            allowDefendFallback; // 无法执行时是否降级为防御
    bool            quickCastFirst;      // 兼容：spellSlotCount>0 时为 true
    int8_t          spellSlot;           // 兼容镜像：等于 spellSlots[0]（1-9/0）
    int8_t          spellSlots[10];      // 循环施法快捷键序列（单行最多 10 槽）
    int8_t          spellSlotCount;      // 有效槽位数 0..10
};
```

### 4.3 方案存储

```cpp
AutoStackRule g_profiles[5][21];   // 5 套已确认方案 × 21 槽位，仅驻留内存
int           g_active_profile;    // 当前生效方案序号
AutoStackRule g_active_rules[21];  // 当前生效规则（执行端读取）
```

### 4.4 当前战斗部队跟踪

玩家提交设置后，执行端会绑定本场人类侧部队跟踪表（`g_stack_track[21]`）：

- **绑定身份**：`side + army_slot_ix + creature_id`
- **运行刷新**：存活（`count_current>0`）、位置（`hex_ix`）、数量（`count_current`）
- **执行前置**：只有活动单位通过跟踪校验时才套用 `g_active_rules[slot]`
- **失效条件**：死亡、槽位被复用/召唤替换、非人类侧、未绑定

这样避免“只按槽位索引取规则”导致的错套：敌方同槽、死亡后槽复用、跨侧活动单位。

## 5. 行动可选集（源：`CellControl_GetAllowedActions`）

按部队类型返回允许的行动列表（决定行动下拉显示哪些项）：

- **普通近战部队**：手动、防御、循环移动、循环近战。
- **射手部队**（`is_ranged`）：手动、防御、循环移动、循环近战、远程攻击。
- **战争机器**：按 §2 返回行动下拉选项（有技能则含「手动」；无技能则仅该机器唯一自动行动）。**不是**启用/禁用开关。

> `is_ranged` 由面板打开时按 `stack.info.shooter` 精确判定并存入 `CellData.is_ranged`；切换方案时复用，不用 `creature_type` 粗判（否则射手如大法师丢失远程选项）。

---

## 6. 目标可选集（源：`CellControl_GetAllowedSelectors`）

按行动类型返回允许的选择器（决定选择器下拉显示哪些项）：

- **远程攻击**：**随机 / 远程高速优先 / 数量最多**。
- **急救治疗**：**失血数值 / 失血比例 / 随机**（候选仅己方伤员；默认失血数值）。
- **循环近战 / 循环移动**：不显示选择器菜单。
- **投石车**：无独立行动枚举；UI 仅手动 / 防御。

---

## 7. 规则规范化（源：`CellControl_NormalizeRule`）

行动变化或加载方案时调用，保证规则自洽：

1. 行动改变时，若当前选择器不在允许集，吸附到允许集首项。
2. 循环近战：压紧并校验 `meleeStandHexes/meleeAttackHexes`，每组必须为相邻格；旧版单组字段首次加载时迁移为序列第 1 组。
3. 循环移动：确保 `moveWaypoints` 已初始化，`kind=AT_POSITION`。
4. 切换到无目标的行动（防御 / 等待）时，清空目标相关字段。

## 8. 设置面板 UI（源：`modules/SettingsDlg.inc.cpp` + `CellControl.inc.cpp`）

### 8.0 模态方案：加固伪模态（不改真模态）

**背景**：曾考虑改为原版 `H3Dlg` 真模态窗口，但真模态靠嵌套阻塞消息循环独占输入，与战场拾取（拾取时需战斗界面响应点击）机制冲突——真模态期间战场被冻结，点不动；要拾取只能关窗重开，体验更差。

**结论**：保留当前**伪模态**方案（自绘到 `screenPcx16` + 自建 hook `Hook_BattleMsgProc`），重点加固：

- 拾取期间确保背景战斗界面不响应误点（屏蔽 hover / 点击穿透）。
- 模态遮挡干净（`ForcePanelModalDepth_` 顶模态计数器）。
- 重绘时机正确（拾取进入 / 退出、格子展开 / 收起边界）。

### 8.1 布局：一行一格宽卡片（568×110）

单列纵向排列，每支部队一张宽卡片。金框恢复 **624×342**，框内刚好 **3 行**（行高按金框反算），超出用滚动条。

**普通部队卡：**

```text
[图标]  行动前循环施法: [1▾][2▾]…＋     （顶部整行，单数字槽，最多 10）
 58×64  数量 12    行动 [远程攻击▾]      选择器 [数量最多▾]
        位置 A07   允许降级 ☑            （近战: 5×2 组合槽 / 移动: 8×2 路径点槽）
```

**战争机器卡**（准入见 §2；弹药车不进面板）：

```text
[图标]  数量 1        行动 [手动/远程/急救/防御▾]   （按机器与技能分支）
 58×64  位置 K08      （无选择器时第三列可空；有急救/远程则显示选择器）
```

战争机器卡（当前实现）：
- **隐藏降级复选框**（`ActionShowsFallback` 对战争机器恒 false）。
- **行动仍是下拉框**（不是启用/禁用开关）。
- **循环施法行仍绘制**（与普通部队同布局；战争机器一般无施法需求，可空着）。
- 位置虽固定但保留显示供参考。

### 8.2 普通部队卡小列布局

- **图标小列**（x=4，58×64）：部队 DEF 图标 + 金框（垂直居中于 110 高卡片）。
- **位置/数量小列**（x=68，宽 74）：数量（右对齐）、位置（贴格子下边缘左对齐）。
- **顶部整行**（y=6）：标签「行动前循环施法:」+ 单行最多 10 个快捷键小槽（已有项可点开重设，末尾「＋」追加；点后弹模态框按 1-9/0 录入）。
- **行动小列**（x=148，宽 202）：行动下拉（中行）+ 允许降级复选框（下行）。
- **第三小列**（x≈192，宽≈368）：选择器下拉，或循环近战 5×2 组合槽，或循环移动 8×2 路径点槽；槽宽按文本宽度定，一行能摆几个就是几个。

### 8.3 下拉展开几何

- 各下拉从其所在小列的行下边缘展开，宽度取所在小列宽（循环施法槽用槽宽）。
- `GetExpandListLeftX / GetExpandListTopY / GetExpandListWidth` 统一计算基准，绘制（`DrawExpandedTo`）、命中（`HitExpandVisibleIndex`）共用。

### 8.4 战场拾取（循环近战 / 移动路径）

统一由透明 BattleUI item 作为输入屏障：从消息中的游戏逻辑坐标调用 `SquareAtCoordinates`（`0x464380`）转换当前 hex，并对点击执行 `StopProcessing()`，不使用坐标下拉，也不让点击穿透到真实战斗操作。

- **循环近战**：点击已有组合或末尾「＋」后隐藏面板；第一击选择站立格并用 `H3CombatManager::ShadeSquare` / `CCellShd` 临时标示，第二击选择与其不同且相邻的攻击目标格。成功后覆盖或追加记录、撤销标示并恢复面板；第二格相同或不相邻时保持拾取；右键取消且不改原记录。
- **循环移动路径**：点「＋」或已有点后进入拾取，只点 1 格写回对应槽（最多 16）；右键取消不改原记录。
- 面板正常打开及拾取期间均保持战场 hover 屏蔽，避免显示待行动部队的移动范围或攻击提示；设置拾取只显示插件自己的单格临时标示。

### 8.5 标签来源（`LoadLabels_`）

INI `[Actions]/[Selectors]` 段提供 UI 标签，严格按枚举下标排列。缺失时用内置默认标签数组兜底。

### 8.6 帮助说明（右上角 `?`）

- 设置面板右上角绘制 `?` 按钮。
- 点击后弹出与快捷施法录入同风格的**伪模态**说明框（遮暗面板 + 金框），内容覆盖：打开方式、5 套方案、循环施法/近战/移动、右键删除、允许降级、人工接管热键、设置生命周期。
- 点「关闭」或按 `ESC` 关闭说明；说明打开时吞掉底层点击，不关闭设置面板本身。

## 9. 方案保存流程

目标行为：打开界面自动加载上次设置；调整过程中不影响已保存设置；仅点「确定（勾✓）」才真正写入。

- 打开面板：`g_profiles → s_p.draft_rules`，进入草稿副本编辑，原方案不动。
- 编辑格子：改动只写 `s_p.cells[].data.rule`（内存草稿）。
- 切换方案（`SelectProfile_`）：先 `SaveCurrentCellsToDraft_` 存回草稿，再 `LoadSelectedProfileIntoCells_` 载入目标方案。
- 点确定（`CommitAndCloseSettingsPanel_`）：`SaveCurrentCellsToDraft_` → `CommitProfiles` 一次性提交全部 5 套到 `g_profiles`。
- 点取消 / ESC：直接关闭，草稿丢弃。

### 9.1 设置生命周期（锚定「一场战斗」）

设置只对**当前这场战斗**有效，生命周期以「玩家接受战斗结果」为终点，而非战斗动画打完：

1. 设置在面板确定后写入 `g_profiles` / `g_active_rules`，整场（含重打）常驻内存。
2. **重打不算新战斗**：结果窗出现后点 HD「取消/重打」（控件 id `0x1FB`）→ 方案保留，BattleUI 回来后 `EnsureStackTrackingBound` 重绑跟踪。
3. 反复重打仍是同一场，设置一路保留。
4. **点结果窗「确定/接受」**（控件 id `0x7802` / iOkay）→ `OnBattleResultAccepted`：清空 5 套方案 + 运行时状态。
5. 无取消按钮的纯原版结果窗：结果窗消失且未点取消 → 视为接受并清除。
6. 实现：每帧检测虚表 `0x0063D46C`（CPResult）的出现/消失与按钮命中（`CheckBattleResultLifecycle_`）。

## 10. 执行分发（源：`modules/AutoExecute.inc.cpp`）

### 10.1 接管判定（`DecideTakeover`）

回放/隐藏战斗（`IsHiddenBattle`）一律 `CD_KEEP_ORIGINAL`。战术布阵阶段不接管。

**最高优先级：人工控制门**（不改 5 套方案、不推进任何循环游标）：

| 模式 | 热键（INI `[Hotkeys]`，可改） | 行为 |
| --- | --- | --- |
| 全手动 | `ToggleManual=F11` 切换 | 本场所有已配置部队都把控制权交回玩家，直到再按一次恢复自动 |
| 单次接管 | `OneShotManual=LControl` 边沿触发 | 锁定当前（或下一支）人类侧部队；该部队完成一次**玩家**动作后自动恢复 |

约束：

- 若当前已有插件提交中的施法/近战等“飞行中”动作，单次接管延后到下一支，绝不半途打断。
- 单次接管绑定 `side+slot+creature`，松键不会丢锁定；真正消费以 `0x4786B0` 执行玩家动作为准。
- 设置面板打开时不响应这两枚热键。
- 状态提示：`打铁助手: 自动 / 全手动 / 单次接管 / 单次待命`。

否则按单位类型判定：

| 单位    | 判定                        |
| ----- | ------------------------- |
| 弹药车   | 保持原版                      |
| 急救帐篷  | 配了急救→执行；否则有急救术→原版，无→交 AI  |
| 投石车   | 防御→代为防御；其余保持原版（手动） |
| 弩车/箭塔 | 远程→执行；有炮术且手动→原版；其余交 AI |
| 普通部队  | 手动→保持原版；否则按配置执行（绝不交回 AI） |

### 10.2 动作提交（`TrySubmitConfiguredAction_`）

按 `rule.action` 分发到 `Submit*`；普通部队失败且允许时降级 `SubmitDefend_`：

- **防御** `SubmitDefend_`：`action=BA_DEFEND`。
- **等待** `SubmitWait_`：`action=BA_WAIT`。
- **循环移动** `SubmitMove_`：按 `moveWaypoints` + `moveWaypointCursor` 逐点巡逻，末点回首点；未配置路径时保留旧单目标兜底。
- **循环近战** `SubmitMelee_`：读取当前 `meleePairCursor` 组合；攻击格上有敌人才出手；`move_type=7`，`action=BA_WALK_ATTACK`，param=站立格，target=攻击格；成功提交后游标推进，失败不推进。
- **远程** `SubmitRanged_`：`SelectStackTarget_` 选敌方目标，`action=BA_SHOOT`。
- **急救** `SubmitFirstAid_`：选己方伤员，`action=BA_FIRST_AID`。
写入后原版主循环（`FUN_004786b0` 等）接管动画 / 伤害 / 回合推进。

### 10.3 目标选择（`SelectStackTarget_`）

先按 `side` 过滤双方 `stack[2][21]` 得候选，再按选择器挑一个：

- `SEL_RANDOM`：随机候选。
- `SEL_RANGED_SPEED`：射手优先，同类按速度降序。
- `SEL_COUNT_HIGH`：存活数量最多。
- `SEL_WOUND_RATIO`：失血比例最高（`失血数值 / (开战数量×单兵满血)`）。
- `SEL_WOUND_VALUE`：失血数值最大（阵亡单位满血 + 当前顶层已损 HP）。

### 10.4 快捷施法执行

**实现**：UI 为 `spellSlots[10]` + `spellSlotCount`（兼容旧 `spellSlot` / `quickCastFirst`）；运行游标在跟踪表 `spell_cursor`。执行端两阶段：
1. 控制权交给玩家时，若序列非空且本回合英雄未施过法，向游戏投递快捷键 `1-9/0`；
2. 观察到 `heroCasted` 变化 / 法力下降 / 超时后推进游标，再提交该部队主动作（主动作为手动时仅施法后交回玩家）。

## 14. 构建与部署

- **统一入口**：`build_and_deploy.bat`（编译 + 部署一步完成，不要单独跑 `build.bat` / `deploy.bat`）。
- 源文件 UTF-8，中文游戏字体走 GBK；换行 CRLF。
- 资源尺寸严格校验：卡片 / 金框尺寸改动后必须用 `tools/gen_*.js` 重生对应 PCX，否则背景加载失败。
- DLL 在游戏进程启动时加载，改动后必须**完全重启游戏**才生效。
